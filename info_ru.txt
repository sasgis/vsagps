Библиотека состоит из следующих логических частей, выделеных в подпапки:
1. DLL - проект для сборки VSAGPS.dll (далее DLL).
2. PUBLIC - публичный интерфейс к DLL.
3. RUNTIME - исполняемая часть (не используется совместно с DLL).
4. SAMPLES - примеры использования библиотеки с исходными кодами.
5. BIN и DCU - для бинарных файлов соответственно.
6. Этот файл, а также любые прочие информационные файлы, не используемые при сборке 
   DLL или прочего ПО, использующего Библиотеку (кроме примеров использования).
7. Файл vsagps_nmea.ini - определения расширений NMEA-0183 для некоторых типов приёмников GPS.

Исключения и дополнения:

1.

Информация об изменениях в исходном коде должна быть отражена в начале изменяемого файла 
исходного кода. Для этого предусмотрено второе поле многострочного комментария. Отдельные записи 
должны быть визуально разделимыми (разделителем не может быть пустая строка или несколько 
пустых строк), должны содержать дату модификации и краткое описание сути выполненных изменений. 
Записи выполняются в обратном хронологическом порядке (новые добавляются в начало).

2.

Запрещается изменять название Библиотеки или DLL.

3.

Запрещается вносить изменения в этот файл.

4.

Запрещается при распространении Библиотеки включать в неё файлы, не имеющие непосредственное 
отношение к Библиотеке и служащие только для увеличения её размера.

5.

При распространении Библиотеки рекомендуется публиковать её в виде архива.

6.

Разрешается добавлять в SAMPLES собственные небольшие примеры использования Библиотеки, кроме 
случаев, когда это существенно увеличивает размер Библиотеки.

7.

Запрещается публиковать Библиотеку на ресурсах, к которым возможен только платный доступ.

8.

При распространении (в том числе модифицированной версии) Библиотеки распространитель обязан 
сохранить в Библиотеке этот файл, дополнив его кратким описанием сути изменений, вместе с тем 
достаточным для понимания в целом без обращения к отдельным исходным файлам. Добавление информации 
необходимо осуществлять аналогично пункту 2 в разделе "История изменений" (ниже), желательно 
с сохранением контактной информации о разработчике, проекте или предприятии, для возможной

из них (на языке файла, в крайнем случае английском или русском языке).




Для целей создания бесплатного ПО, лицензируемого в соответствии с лицензией, несовместимой 
с настоящей Лицензией, Библиотека дополнительно лицензируется в "урезанном" варианте (независимо

DLL вместе с ПО, которое её использует. Запрещается любым образом модифицировать Библиотеку,

в явном виде независимо от того, выполняется ПО или нет. Также запрещается скачивать DLL в процессе 
работы ПО (кроме целей автоматического обновления ПО при выпуске Автором новой версии Библиотеки).
